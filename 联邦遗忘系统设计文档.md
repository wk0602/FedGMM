## 系统概述

本系统实现了一个**联邦学习环境下的模型遗忘（Unlearning）框架**，专门用于移除联邦模型中特定客户端数据的影响，特别是针对**后门攻击（Backdoor Attack）**场景。

### 核心问题

在联邦学习中，当某个客户端的数据需要被"遗忘"时（例如数据隐私撤回、恶意数据清理），传统的做法是重新训练整个模型，这在联邦环境下成本极高。本系统通过**选择性参数更新**和**对抗性遗忘损失**，实现了高效的联邦遗忘。

### 主要创新

1. **对抗性遗忘损失（Adversarial Forgetting Loss）**：不同于传统的KL散度到均匀分布（产生"不确定"预测），本系统强制模型对遗忘数据做出**"自信但错误"**的预测，从而更彻底地打破后门模式关联。

2. **梯度掩码机制（Gradient Masking）**：仅更新对遗忘数据最敏感的Top-ρ参数，减少对保留任务的影响。

3. **可选的梯度散度掩码**：基于遗忘数据和保留数据的梯度差异，智能选择需要遗忘的参数。

## 系统架构

### 类继承关系

```
BasicTask (基础任务类)
    └── UnlearningTask (遗忘任务类)
            └── 管理客户端标记、后门注入、测试评估

Algorithm (基础算法类)
    └── UnlearnAlgorithm (遗忘算法基类)
            ├── FedGMM (原始FedGMM算法)
            └── FedGMM_Adversarial (对抗性FedGMM算法)
```

### 核心组件

#### 1. UnlearningTask（任务层）

**职责**：
- 客户端标记：将指定客户端标记为`unlearn_flag=True`
- 后门注入：为遗忘客户端注入后门模式（用于测试遗忘效果）
- 测试评估：维护本地测试准确率和后门攻击成功率（ASR）
- 参数管理：解析和配置遗忘相关参数

**关键方法**：
- `modify_client()`: 为客户端注入后门数据
- `ClientTest.run()`: 评估客户端模型性能（正常准确率和ASR）
- `outFunc()`: 输出遗忘和保留客户端的性能统计

#### 2. UnlearnAlgorithm（算法基类）

**职责**：
- 提供遗忘算法的通用框架
- 实现早停机制（基于ASR）
- 跟踪模型参数变化（距离初始模型的距离）
- 管理遗忘轮数限制

**关键特性**：
- **早停机制**：当ASR低于阈值或连续N轮无改善时自动停止
- **参数跟踪**：记录模型参数与初始模型的距离
- **冲突检测**：统计梯度冲突角度（用于分析遗忘效果）

#### 3. FedGMM_Adversarial（核心算法）

**职责**：
- 实现对抗性遗忘损失
- 管理梯度掩码的生成和刷新
- 执行带掩码的梯度聚合
- 监控遗忘效果（正确类别概率）

## 核心算法：FedGMM-Adversarial

### 算法流程

```
1. 初始化阶段
   ├── 创建对抗性遗忘损失函数
   ├── 为遗忘客户端分配损失函数
   └── 生成初始梯度掩码

2. 遗忘循环（每轮）
   ├── 客户端本地训练（使用对抗性损失）
   ├── 收集本地梯度
   ├── 刷新掩码（可选，周期性）
   ├── 应用掩码到遗忘客户端梯度
   ├── 加权聚合所有梯度
   └── 更新全局模型

3. 终止条件
   ├── ASR <= 阈值（早停）
   ├── 连续N轮无改善（早停）
   └── 达到最大遗忘轮数
```

### 算法伪代码

```python
# 初始化
adversarial_loss = AdversarialForgettingLoss(...)
for client in clients:
    if client.unlearn_flag:
        client.criterion = adversarial_loss
masks = build_masks()  # 基于梯度幅度或梯度散度

# 主循环
while not terminated():
    # 客户端本地训练
    gradients = []
    for client in online_clients:
        grad = client.train_local()  # 使用对抗性损失
        gradients.append(grad)
    
    # 应用掩码
    masked_gradients = []
    for grad, client in zip(gradients, online_clients):
        if client.unlearn_flag:
            grad = grad * masks[client.id]  # 仅保留敏感参数
        masked_gradients.append(grad)
    
    # 加权聚合
    global_grad = weighted_average(masked_gradients)
    
    # 更新模型
    model = model - lr * global_grad
    
    # 刷新掩码（周期性）
    if should_refresh_mask():
        masks = refresh_masks()
    
    # 检查早停
    if check_early_stopping():
        break
```

## 损失函数设计

### 设计理念

传统遗忘方法（如FedGMM）使用**KL散度到均匀分布**，使模型对遗忘数据产生**不确定预测**（所有类别概率接近1/K）。这种方法的问题是：

- **软遗忘**：模型仍然"知道"这些数据，只是不确定如何分类
- **后门残留**：后门模式关联可能仍然存在

**对抗性遗忘损失**的核心思想是：**强制模型对遗忘数据做出自信但错误的预测**，从而更彻底地打破模式关联。

### 数学公式

#### 1. Confident Wrong Loss（推荐模式）

```
L_adversarial = -log(1 - p_correct + ε) - λ_entropy * H(p)

其中：
- p_correct: 模型对正确类别的预测概率
- H(p): 预测分布的熵
- λ_entropy: 熵正则化权重
- ε: 数值稳定性常数
```

**性质**：
- 当 `p_correct → 1`（自信正确）：损失 → +∞（严重惩罚）
- 当 `p_correct → 0`（自信错误）：损失 → 0（奖励）
- 熵项防止模型崩溃到单一错误类别

#### 2. Margin-Based Loss（稳定模式）

```
L_margin = ReLU(p_correct - margin)² - λ_entropy * H(p)

其中：
- margin: 阈值（默认0.3），当p_correct < margin时不惩罚
- 仅在模型过于自信正确时才应用损失
```

#### 3. Hybrid Loss（平衡模式）

```
L_hybrid = 0.7 * L_confident_wrong + 0.3 * L_KL_uniform

结合对抗性损失和KL-uniform损失，提供平衡的遗忘效果
```

### 损失函数实现

```python
class AdversarialForgettingLoss:
    def __call__(self, pred, target):
        # 应用温度缩放
        scaled_logits = pred / self.temperature
        probs = F.softmax(scaled_logits, dim=-1)
        
        # 获取正确类别的概率
        correct_probs = probs[range(batch_size), target]
        
        # 对抗性损失：最小化正确类别概率
        adversarial_loss = -log(1.0 - correct_probs + eps)
        
        # 熵正则化：鼓励多样化的错误预测
        entropy = -sum(probs * log_probs, dim=-1)
        
        # 总损失
        total_loss = adversarial_loss - entropy_weight * entropy
        
        return weight * total_loss.mean()
```

## 梯度掩码机制

### 掩码生成策略

#### 1. 梯度幅度掩码（默认）

**原理**：选择对遗忘数据梯度幅度最大的Top-ρ参数

```python
def compute_gradient_magnitude_mask(grad_vec):
    k = int(len(grad_vec) * rho)  # rho: 掩码比例（默认0.15）
    _, indices = topk(abs(grad_vec), k=k)
    mask = zeros_like(grad_vec)
    mask[indices] = 1.0
    return mask
```

**优点**：
- 计算简单高效
- 直接识别对遗忘数据最敏感的参数

**缺点**：
- 可能选择对保留任务也重要的参数

#### 2. 梯度散度掩码（可选）

**原理**：选择在遗忘数据和保留数据上梯度方向差异最大的参数

```python
def compute_gradient_divergence_mask(unlearn_client):
    # 计算遗忘数据梯度
    forget_grad = compute_gradient(unlearn_client)
    
    # 计算保留数据平均梯度
    retain_grads = [compute_gradient(c) for c in retain_clients]
    retain_grad_avg = mean(retain_grads)
    
    # 计算散度分数
    # 高散度 = 梯度方向相反 = 对后门重要但对正常任务不重要
    divergence = -forget_grad * retain_grad_avg + abs(forget_grad)
    
    # 选择Top-k最散度的参数
    k = int(len(divergence) * rho)
    _, indices = topk(divergence, k=k)
    mask = zeros_like(divergence)
    mask[indices] = 1.0
    return mask
```

**优点**：
- 更智能地识别后门特定参数
- 减少对保留任务的影响

**缺点**：
- 需要计算多个客户端的梯度，计算成本更高

### 掩码刷新策略

掩码可以**静态**（仅初始化时计算）或**动态**（周期性刷新）：

```python
# 刷新间隔（adv_mask_refresh）
if current_round % mask_refresh == 0:
    masks = refresh_masks()  # 重新计算掩码
```

**刷新原因**：
- 随着模型更新，参数的敏感度会变化
- 动态掩码能更好地适应遗忘过程

## 系统特性

### 1. 早停机制

**目的**：当遗忘效果达到目标时自动停止，节省计算资源

**触发条件**：
1. **ASR阈值**：当攻击成功率（ASR）≤ 阈值（默认0.05）时停止
2. **无改善停止**：连续N轮（默认10轮）ASR无显著改善时停止

**实现**：
```python
def check_early_stopping():
    current_asr = mean([client.backdoor_test_accuracy for client in unlearn_clients])
    
    # 条件1：ASR低于阈值
    if current_asr <= threshold:
        return True
    
    # 条件2：无改善
    if current_asr < best_asr - min_delta:
        best_asr = current_asr
        patience_counter = 0
    else:
        patience_counter += 1
    
    if patience_counter >= patience:
        return True
    
    return False
```

### 2. 性能监控

系统自动跟踪以下指标：

- **正确类别概率**：遗忘数据上模型对正确类别的平均预测概率（应随时间下降）
- **ASR（攻击成功率）**：后门测试数据的准确率（应随时间下降）
- **保留准确率**：保留客户端上的正常测试准确率（应保持稳定）
- **模型距离**：当前模型参数与初始模型的距离

### 3. 梯度冲突检测

系统统计遗忘客户端梯度与聚合梯度的冲突角度：

```python
conflict_angle = arccos(dot(grad_local, grad_global) / (norm(grad_local) * norm(grad_global)))
```

高冲突角度表明遗忘客户端与其他客户端的学习目标存在冲突，这是遗忘过程的正常现象。

## 配置与使用

### 配置文件结构

系统支持YAML配置文件，参数优先级：**命令行参数 > 配置文件 > 默认值**

#### 基础配置（config_adversarial.yaml）


## 技术细节

### 1. 客户端标记机制

系统通过`unlearn_flag`属性标记需要遗忘的客户端：

```python
# 在UnlearningTask中
for client in algorithm.client_list:
    if client.id in unlearn_client_id_list:
        setattr(client, "unlearn_flag", True)
        modify_client(client, attack_portion)  # 注入后门
    else:
        setattr(client, "unlearn_flag", False)
```

### 2. 损失函数分配

遗忘客户端使用对抗性损失，保留客户端使用标准交叉熵损失：

```python
# 在FedGMM_Adversarial.run()中
adversarial_loss = AdversarialForgettingLoss(...)
for client in client_list:
    if client.unlearn_flag:
        client.criterion = adversarial_loss
    # 保留客户端保持默认的CrossEntropyLoss
```

### 3. 梯度聚合

系统使用**加权平均**聚合梯度，权重基于客户端本地数据量：

```python
weights = Tensor([client.local_training_number for client in clients])
weights = weights / weights.sum()  # 归一化
global_grad = weights @ masked_gradients  # 加权平均
```

### 4. 模型更新

使用标准的梯度下降更新：

```python
model_params = model_params - lr * global_grad
```

### 5. 后门注入

系统使用`FigRandBackdoor`为遗忘客户端注入后门模式：

```python
backdoor = FigRandBackdoor(...)
backdoor.add_backdoor(client.local_training_data, attack_portion=0.8)
```

后门模式用于测试遗忘效果（ASR指标）。

## 算法对比

### FedGMM vs FedGMM-Adversarial

| 特性 | FedGMM | FedGMM-Adversarial |
|------|--------|-------------------|
| **损失函数** | KL散度到均匀分布 | 对抗性遗忘损失 |
| **遗忘策略** | 使预测不确定（p→1/K） | 使预测自信错误（p_correct→0） |
| **遗忘强度** | 软遗忘 | 硬遗忘 |
| **后门移除** | 可能残留 | 更彻底 |
| **掩码比例** | 较小（0.05） | 较大（0.15） |
| **掩码刷新** | 通常不刷新 | 周期性刷新 |
| **计算成本** | 较低 | 中等（可选梯度散度） |

### 损失函数对比

#### KL-Uniform Loss（FedGMM）

```
L_KL = -sum(uniform * log_probs)
     = -sum((1/K) * log_probs)
     
效果：p → [1/K, 1/K, ..., 1/K]  # 均匀分布
```

#### Adversarial Loss（FedGMM-Adversarial）

```
L_adv = -log(1 - p_correct) - λ * entropy

效果：p_correct → 0, p_others → 高值  # 自信错误
```

### 适用场景

**FedGMM**适用于：
- 需要温和遗忘的场景
- 计算资源受限
- 对保留任务影响要求极低

**FedGMM-Adversarial**适用于：
- 需要彻底移除后门攻击
- 对遗忘效果要求高
- 可以接受稍高的计算成本

## 总结

本联邦遗忘系统通过**对抗性遗忘损失**和**梯度掩码机制**，实现了高效的联邦模型遗忘。核心创新在于：

1. **对抗性遗忘损失**：强制模型对遗忘数据做出自信但错误的预测，比传统的不确定预测更彻底地打破模式关联。

2. **智能梯度掩码**：仅更新对遗忘数据最敏感的参数，减少对保留任务的影响。

3. **自适应机制**：支持掩码刷新、早停、梯度散度等高级特性，提高遗忘效果和效率。

系统设计兼顾了**遗忘效果**、**保留任务性能**和**计算效率**，为联邦学习环境下的模型遗忘提供了实用的解决方案。
